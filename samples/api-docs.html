PageIndex API Documentation

A vectorless RAG solution for document indexing and retrieval

Introduction

PageIndex is a revolutionary approach to document retrieval that uses hierarchical tree structures and LLM reasoning instead of traditional vector embeddings. This approach offers several advantages:

  • No vector database required
  • Interpretable search process
  • Better handling of document structure
  • Reduced storage requirements

Installation

Using npm

npm install pageindex-vectorless

Using yarn

yarn add pageindex-vectorless

Quick Start

Get started with PageIndex in just a few lines of code:


import { PageIndex } from 'pageindex-vectorless';

const pageIndex = new PageIndex({
    openaiApiKey: process.env.OPENAI_API_KEY
});

// Index a document
const doc = await pageIndex.indexFromFile('document.pdf');

// Search the document
const results = await pageIndex.query('What is the main topic?');

API Reference

PageIndex Class

The main class for document indexing and querying.

Constructor

new PageIndex(config: PageIndexConfig)
Parameter Type Description
openaiApiKey string Your OpenAI API key
openaiModel string Model to use (default: gpt-4o-mini)
dataDir string Directory for storing indexes

Indexing Methods

indexFromFile(filePath, options)

Index a document from a local file path.


const doc = await pageIndex.indexFromFile('/path/to/document.pdf', {
    name: 'my-document',
    useLLMTreeBuilder: true
});

indexFromUrl(url, options)

Index a document from a URL.


const doc = await pageIndex.indexFromUrl('https://example.com/doc.pdf');

indexContent(content, name, contentType)

Index content directly.


const doc = await pageIndex.indexContent(buffer, 'doc-name', 'text/markdown');

Search Methods

query(query, documentIds, options)

Search indexed documents using LLM reasoning.


const results = await pageIndex.query(
    'What are the key features?',
    ['doc-id-1', 'doc-id-2'],
    { topK: 5, includeReasoning: true }
);
SearchOptions
Option Type Default Description
topK number 3 Number of results to return
maxIterations number 5 Max tree traversal iterations
includeReasoning boolean true Include LLM reasoning in results

Supported Document Formats

  • PDF - Portable Document Format (.pdf)
  • HTML - HyperText Markup Language (.html, .htm)
  • Markdown - Markdown files (.md)
  • Plain Text - Text files (.txt)

Architecture

PageIndex uses a three-stage process:

  1. Parsing - Documents are parsed into structured sections
  2. Tree Building - Sections are organized into a hierarchical tree
  3. LLM Search - Queries are answered by traversing the tree with LLM reasoning

Tree Structure

Each node in the tree contains:

  • node_id - Unique identifier
  • title - Section title
  • summary - Content summary
  • start_index - Starting page/position
  • end_index - Ending page/position
  • nodes - Child nodes

Examples

Index Multiple Documents


const docs = ['doc1.pdf', 'doc2.md', 'doc3.html'];
for (const doc of docs) {
    await pageIndex.indexFromFile(doc);
}

Search with Specific Documents


const results = await pageIndex.query(
    'company revenue',
    ['financial-report', 'quarterly-earnings']
);

Get Document List


const documents = await pageIndex.listDocuments();
console.log(documents);
